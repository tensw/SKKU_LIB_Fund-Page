## 1. 프로젝트 배경 및 목적

### 1.1 사업 배경

성균관대학교 학술정보관은 **1398 LibraryON 프로젝트**를 통해 총 사업비 **50억 원** 규모의
도서관 리뉴얼을 추진하고 있습니다. 이 프로젝트는 삼성학술정보관(자연과학캠퍼스)과
중앙학술정보관(인문사회과학캠퍼스) 두 캠퍼스를 대상으로 합니다.

"1398"은 성균관대학교의 설립 연도(1398년)를 의미하며, "LibraryON"은
**전통을 켜다(ON), 지식을 ON하다, 24시간 살아있는 도서관**이라는 세 가지 의미를 담고 있습니다.

### 1.2 시스템 목적

| 목적 | 설명 |
|------|------|
| **기부 독려** | 발전기금 모금 현황을 투명하게 공개하여 기부 참여를 유도 |
| **기부자 예우** | 기부자 명단과 등급별 예우 혜택을 공개하여 감사를 표현 |
| **프로젝트 홍보** | 1398 LibraryON 프로젝트의 비전과 방향을 소개 |
| **데이터 관리** | 관리자가 기부 데이터를 효율적으로 등록·조회·분석 |

### 1.3 대상 사용자

| 사용자 | 역할 | 주요 행위 |
|--------|------|-----------|
| **예비 기부자** | 학생, 교직원, 동문, 일반인 | 프로젝트 확인 → 기부 현황 열람 → 기부 참여 (외부 결제) |
| **기존 기부자** | 이미 기부한 사용자 | 자신의 명단 확인, 예우 혜택 확인 |
| **학술정보관 직원** | 관리자 | 기부 데이터 등록(엑셀), 현황 분석, 다운로드 |
| **대학 관계자** | 보고 열람 | 대시보드를 통한 기부 현황 파악 |

### 2.2 사용자 페이지 기능 상세

#### A. 히어로 섹션
- **의도**: 첫 화면에서 프로젝트의 정체성과 기부 참여 동선을 즉시 제공
- **구성**: 풀스크린 배경 이미지 + "1398 LibraryON" 타이포그래피 + **발전기금 기부참여** CTA 버튼
- **CTA 링크**: `give.skku.edu` → 1398캠페인 > 1398LibraryOn(도서관)발전기금
- **활용**: 학술정보관 홈페이지, 교내 포털, SNS 등에서 공유 시 첫 인상을 결정하는 핵심 영역

#### B. 프로젝트 비전 소개
- **의도**: 기부금이 어디에 쓰이는지 구체적으로 보여주어 기부 동기를 부여
- **구성**:
  - 3개 비전 카드: **스마트 도서관** (AI 기반) / **멀티 학습 공간** (협업) / **문화·예술 체험** (열린 공간)
  - 이미지 갤러리: 사랑해 라운지, 멀티 학습 공간, 스마트 도서관 컨셉 이미지
  - 총 사업비 50억 원 표시 (삼성학술정보관 + 중앙학술정보관)
- **활용**: 기부자에게 "나의 기부가 만드는 변화"를 시각적으로 전달

#### C. 기부 현황 (실시간)
- **의도**: 투명한 모금 현황 공개로 신뢰 확보, 참여 건수를 강조하여 "함께하는 느낌" 전달
- **구성**:
  - **참여 건수** / **기부 총액**: 카운트업 애니메이션 (스크롤 시 작동, 시각적 임팩트)
  - 프로젝트별 상세 카드: 1398 LibraryON / 도서관 발전기금 각각의 금액·건수
- **데이터 흐름**: `/api/donations/summary` API에서 실시간 집계 데이터 조회
- **활용**: 모금 목표 대비 달성률을 시각적으로 보여주어 추가 기부 독려

#### D. 기부자 명단
- **의도**: 기부자에 대한 공식적 감사 표현, 기부 문화 확산
- **구성**:
  - 테이블: No., 기부자, 기부자 정보(유형), 기부금액, 기부일, 비고(용도)
  - **검색**: 기부자 이름 / 기부자 정보(유형)로 검색
  - **필터**: 기부 용도 (도서관 발전기금 / 1398 LibraryON)
  - **정렬**: 기부금액 오름차순 / 내림차순
  - **페이징**: 10건 단위, 최대 5개 페이지 번호 표시
  - **엑셀 다운로드**: 전체 기부 내역을 .xlsx 파일로 내보내기
- **데이터 흐름**: `/api/donations` API (서버 사이드 페이징/필터/정렬)
- **활용**: 학술정보관 내부 보고, 기부자 확인, 감사장 발송 대상 추출

#### E. 기부자 예우 안내
- **의도**: 기부 금액에 따른 혜택을 명확히 안내하여 상위 등급 기부 유도
- **구성**: 4단계 등급 카드

| 등급 | 기준 | 주요 혜택 |
|------|------|-----------|
| 일반 기부자 | 1만원 이상 | 감사 서신, 명단 등재, 소식지 발송 |
| 우수 기부자 | 100만원 이상 | + 특별 열람 서비스, 행사 초청, 감사패 |
| 명예 기부자 | 1,000만원 이상 | + 명예의 전당, 총장 감사장, VIP 열람실 |
| 공간명 헌정 | 사업비 50% 이상 | + 공간명 명명, 영구 감사 명판, 자문위원 위촉 |

- **활용**: 기부자 상담 시 혜택 안내 자료로 활용, 고액 기부 유치 근거

#### F. 기부 참여 안내 + 연락처
- **의도**: 관심을 가진 사용자가 즉시 행동으로 옮길 수 있는 동선 제공
- **구성**:
  - **CTA 영역**: "발전기금 기부참여" 버튼 (give.skku.edu 연결) + "기부참여방법 안내" 버튼
  - **연락처**: 학술정보관 (02-760-1207, 031-299-4021) / 발전협력팀 (02-760-1153)
  - **이메일**: centrallib@skku.edu, samsunglib@skku.edu, fund@skku.edu
- **활용**: 오프라인 기부 문의, 단체 기부 상담 연결

### 2.3 관리자 페이지 기능 상세

#### A. 로그인
- **의도**: 비인가 접근 차단
- **방식**: 비밀번호 기반 인증 → httpOnly Cookie 발급 (24시간 유효)
- **보호 범위**: `/admin/*` 전체 경로를 미들웨어로 보호

#### B. 대시보드
- **의도**: 기부 현황을 한눈에 파악할 수 있는 경영 정보 제공
- **구성**:

| 요소 | 내용 |
|------|------|
| KPI 카드 4종 | 총 기부 건수, 총 기부액, 평균 기부액, 최근 기부일 |
| 월별 추이 차트 (Line) | 기간별 기부 추세 파악 |
| 용도별 분포 차트 (Pie) | 1398 LibraryON vs 도서관 발전기금 비중 |
| 유형별 현황 차트 (Bar) | 교원/직원/동문/학부/일반 기부 비교 |
| 최근 기부내역 테이블 | 최근 10건 리스트 |
| 기간 필터 | 1개월 / 3개월 / 6개월 / 전체 |

- **활용**: 월간/분기 보고서 작성, 기부 캠페인 효과 분석, 타겟 기부자 그룹 파악

#### C. 엑셀 업로드
- **의도**: 기존 기부 데이터를 일괄 등록, 주기적 데이터 갱신
- **구성**:
  - 파일 선택 (.xlsx, .xls 지원)
  - **미리보기**: 업로드 전 데이터 확인 (최대 50행)
  - **유연한 헤더 매칭**: 한국어/영어 다양한 헤더명 자동 인식
  - 업로드 → DB 저장 → 결과 표시
- **지원 헤더 예시**: "기부자", "기부자명", "name" → 모두 같은 필드로 매핑
- **활용**: 발전협력팀에서 관리하는 엑셀 데이터를 웹 시스템에 주기적 반영

### 2.4 공통 UI

#### 헤더 (글로벌 네비게이션)
- **의도**: 학술정보관 본 사이트(lib.skku.edu)와 동일한 메뉴 구조를 제공하여 통합된 사용자 경험 유지

#### 섹션 네비게이션 (Sticky)
- **의도**: 긴 단일 페이지에서 원하는 섹션으로 빠르게 이동
- **구성**: 안내 / 기부현황 / 기부자 명단 / 기부자 예우 4개 탭
- **동작**: 스크롤 위치에 따라 활성 탭 자동 전환, 클릭 시 부드러운 스크롤

#### 푸터
- **의도**: 법적 필수 정보(개인정보처리방침, 이메일무단수집거부) 및 관련 사이트 링크 제공

## 3. 기술 아키텍처

### 3.1 기술 선정 근거

| 기술 | 선정 이유 |
|------|-----------|
| **Next.js 16** | App Router + API Routes로 프론트/백엔드 통합, 별도 서버 불필요 |
| **React 19** | 최신 서버 컴포넌트 지원, 성숙한 생태계 |
| **Tailwind CSS 4** | 빠른 UI 개발, 일관된 디자인 시스템 |
| **Prisma 6** | 타입 안전한 DB 접근, 자동 클라이언트 생성 |
| **SQLite** | 별도 DB 서버 불필요, 파일 기반으로 배포 간편 |
| **Framer Motion** | 선언적 애니메이션, React 네이티브 통합 |
| **Recharts** | React 기반 차트, 반응형 지원 |
| **SheetJS (xlsx)** | 엑셀 파싱/생성, 브라우저+서버 양쪽 지원 |

### 3.2 데이터 모델

```
Donation (기부 내역)
┌──────────┬──────────┬──────────────────────────────────┐
│ 필드     │ 타입     │ 설명                              │
├──────────┼──────────┼──────────────────────────────────┤
│ id       │ Int (PK) │ 자동 증가 고유 번호                │
│ name     │ String   │ 기부자 이름 (익명기부자 가능)      │
│ donorType│ String   │ 교원/직원/동문/학부/일반            │
│ amount   │ BigInt   │ 기부 금액 (원 단위)                │
│ donatedAt│ DateTime │ 기부 일자                          │
│ purpose  │ String   │ 도서관 발전기금 / 1398 LibraryON   │
│ note     │ String?  │ 비고 (선택)                        │
│ createdAt│ DateTime │ 레코드 생성 시각                   │
└──────────┴──────────┴──────────────────────────────────┘
```

### 3.3 API 설계

| 엔드포인트 | 용도 | 주요 파라미터 |
|-----------|------|--------------|
| `GET /api/donations` | 기부 목록 조회 | page, limit, search, type, purpose, sort |
| `GET /api/donations/summary` | 요약 통계 | - |
| `GET /api/donations/analytics` | 분석 데이터 | period (1m/3m/6m/all) |
| `GET /api/donations/export` | 엑셀 다운로드 | - |
| `POST /api/admin/auth` | 관리자 로그인 | password |
| `DELETE /api/admin/auth` | 관리자 로그아웃 | - |
| `POST /api/admin/upload` | 엑셀 업로드 | file (multipart) |

### 3.4 인증 흐름

```
사용자 → /admin/login → 비밀번호 입력
  → POST /api/admin/auth → 비밀번호 검증
    → 성공: Set-Cookie (admin-token, httpOnly, 24h)
    → 실패: 401 에러 메시지

/admin/* 접근 → middleware.ts
  → Cookie 있음: 통과
  → Cookie 없음: /admin/login 리다이렉트

로그아웃 → DELETE /api/admin/auth → Cookie 삭제
```

## 5. 활용 시나리오

### 5.1 기부 캠페인 시나리오

```
[시작] 학술정보관이 기부 캠페인 공지
  → 교내 포털/SNS에 웹 시스템 URL 공유
    → 예비 기부자가 접속
      → 히어로 섹션에서 프로젝트 파악
      → 기부 현황에서 "이미 N명이 참여" 확인 (사회적 증거)
      → 기부자 예우에서 혜택 확인
      → "발전기금 기부참여" 버튼 클릭
        → give.skku.edu에서 결제 완료

[후속] 관리자가 기부 데이터 엑셀로 정리
  → /admin/upload에서 엑셀 업로드
    → 웹 시스템에 실시간 반영
      → 메인 페이지 기부 현황 숫자 자동 증가
      → 기부자 명단에 새로운 이름 추가
```

### 5.2 정기 보고 시나리오

```
[월말] 관리자가 /admin 대시보드 접속
  → 기간 필터: "1개월" 선택
    → 이번 달 기부 건수, 금액, 추이 확인
    → 유형별 차트: 어떤 그룹이 활발한지 파악
    → 용도별 분포: 프로젝트별 비중 확인
  → /api/donations/export로 엑셀 다운로드
    → 보고서 첨부 자료로 활용
```

### 5.3 기부자 감사 시나리오

```
[분기] 기부자 감사장 발송 대상 추출
  → 기부자 명단에서 금액순 정렬 (내림차순)
  → 엑셀 다운로드 → 감사장 발송 목록 생성
  → 등급별 예우 적용:
    - 1,000만원 이상: 총장 감사장 + VIP 열람실
    - 100만원 이상: 감사패 + 행사 초청
    - 1만원 이상: 감사 서신 + 소식지
```

### 5.4 고액 기부자 유치 시나리오

```
[상시] 잠재 고액 기부자 상담
  → 웹 시스템으로 프로젝트 비전 시각적 설명
  → 기부자 예우 페이지에서 "공간명 헌정" 안내
    → "도서관 공간에 기부자명이 영구 부착됩니다"
  → 현재 기부 현황으로 프로젝트의 활성도 증명
  → 연락처 안내 → 맞춤 상담 연결
```

## 8. 운영 프로세스

### 8.1 데이터 갱신 워크플로

```
발전협력팀에서 기부 데이터 정리 (엑셀)
  ↓
학술정보관 담당자가 /admin/login 접속
  ↓
/admin/upload에서 엑셀 파일 업로드
  ↓
미리보기 확인 → "업로드" 클릭
  ↓
자동으로 메인 페이지에 반영
  (기부 현황 숫자, 기부자 명단 모두 갱신)
```

| 필드 | 필수 | 인식 가능한 헤더명 |
|------|------|-------------------|
| 기부자명 | ✅ | 기부자, 기부자명, 이름, name |
| 기부금액 | ✅ | 기부금액, 기부 금액, 금액, amount |
| 기부자유형 | - | 기부자정보, 기부자 정보, 기부자유형, 유형 |
| 기부일 | - | 기부일, 기부날짜, 기부일자, 날짜 |
| 기부용도 | - | 기부용도, 기부 용도, 용도, purpose |
| 비고 | - | 비고, note, 메모 |
